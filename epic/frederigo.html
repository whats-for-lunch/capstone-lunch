Use Case

Persona.

Name: Frederigo Sanchez

Age: 22.

Gender: Male.

Technology: Macbook Air 13 inch display, running MacOS Mojave 10.14.4 With a 1.8 GHz intel core i5 processor He uses a second HP 24-inch monitor with (1920 x 1080 @60 Hz) Frederigo also uses an iPhone XR with a 6.1 inch LCD display, running IOS 12.2.

Technical Proficiency: Very experienced in the technology he uses throughout the day.

Occupation: Deep-Dive Student.

Title: Pick a place to eat for the indecisive lunch goer.

Desired Features: He wants to be able to save a preferred restaurant in a favorite list.

Description: Frederigo is a DeepDive student and only has an hour for lunch and wants to start a favorites list.

Name of the User and their role Frederigo, a professional student, can't decide on where to go eat for his hour lunch, but he enjoys trying new places and creating lists.

Usage Preconditions: Frederigo already has an account and is signed in.

Usage Postconditions: Frederigo is able to find a restaurant to eat at and add to favorites list on website.

User Story: As a power user I would like to be able to manage my favorite restaurants in a favorites page.

Interaction Flow

  • Server shows map with markers
  • He clicks on a marker of a favorite restaurant
  • opens new window with info of restaurant and an 'add to favorites' option
  • click favorites page
  • server updates the favorite list page
  • Server is updated with highlighted markers
